DS05: Solution hatch_cage

Background-Information

The eggs for the next generation are collected in cages. Multiple cages can be available for each line. The loadingstream DS01-04 defines from which cage eggs are collected for the next generation.

The information stored in the database can be seen in the next view:

The loadingstream DS05 has the following structure

The information in table 'hatch_cage' is retrieved via the date of birth and the concatenated columns C and D (concatenated to 'JH:::1'). With this information the cage with id 176 is found. The animals in there are the parents of animal with id '2243' in DS05.

Now a new animal can be created.

No Parent-Cage found in table 'hatch_cage' => Solutions

1th Solution: Add the missing record in DS04

  • open CageBook for breed (LS) with birthyear of parents (2019) and look for 'Far hens'=stammenummer (1)
  • If there are several cages with the same 'Far hens'-number choose one with counts of eggs.
  • take note the Cage number (176), Breed (LS), 'Far hens' (1), 'Event date' (26.04.2019), Traits (37(35/28)
  • Add a new record with this information in DS01-04
  • reload DS01-04
  • reload DS05

2nd Solution: Add the missing record via form

  • open CageBook for breed (LS) with birthyear of parents (2019) and look for 'Far hens'=stammenummer (1)
  • If there are several cages with the same 'Far hens'-number choose one with counts of eggs.
  • take note the Cage number (176), Breed (LS), 'Far hens' (1), 'Event date' (26.04.2019), Traits (37(35/28)
  • open Web-Frontend => Edit => Hatch Cage

  • reload DS05
